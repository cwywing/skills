# Article template — section-by-section

Read at Step 4 before drafting. This template bakes the five GEO layers
(`geo-principles.md`) into the article's skeleton so they are not an
afterthought. Not every section is mandatory — drop a section only when the
question set (Step 3) shows no real user question maps to it. Never keep a
section that has no information to carry: an empty "FAQ" of filler questions
hurts GEO more than having no FAQ.

## Lead block (always present)

First paragraph, 2–4 sentences. Must contain:

1. The core entity's full canonical name (+ alias / English / abbreviation if
   relevant).
2. A one-line definition that distinguishes it from namesakes.
3. The single most distinguishing verifiable fact (founding year + location,
   or core product + certification, or category + scope).

This is the sentence an AI engine is most likely to lift verbatim. Write it as
a complete, self-contained, citable statement — no bare pronouns ("该系统",
"我们", "该公司") whose referent lives in the title or a prior paragraph,
because the citation may be pulled out of context and lose the referent.
Restate the canonical name instead.

## Definition / "What is X" section

Answers the *identification* questions. Define the entity, its category, and
its scope in plain language. State what it is *and* what category it belongs
to, so an AI can place it in the right taxonomy. Avoid jargon-only definitions
— give the lay reader's version first, then the technical one if the domain
needs it.

## Scope & boundary section

Answers *scope/boundary* questions. What the entity covers, what it does not.
This is where most GEO value lives: an AI summarizer drawing a boundary will
cite the article that draws it clearly. List concrete in-scope and
out-of-scope items rather than abstract claims.

## Evidence / credibility section

Answers *evidence* questions. Credentials, certifications, regulatory
filings, third-party coverage, customer cases (named, not "a Fortune 500
client"). Each item cited to its source. This is the cross-corroboration layer
made literal — the article shows the AI exactly which authorities back which
claims.

## Comparison section (include when the question set has comparison intent)

Answers "X vs Y" / "X 与 Z 有什么区别" queries. Compare on concrete dimensions,
not vibes. A small comparison table often helps here and is highly citable
(engines lift structured data readily). Always define both sides of the
comparison, not just the favored one — a one-sided "we're better" comparison
is not citable.

## FAQ section (include when there are 4+ real user questions not already
answered above)

Each question gets a short, self-contained answer (2–4 sentences). Questions
must be real user phrasings, not invented long-tail keywords. If a question
is already fully answered in an earlier section, do not duplicate — point to
it or drop it. Duplicating an answer with slightly different wording risks
teaching the AI inconsistent phrasings of the same fact.

## Sources block (always present)

List the sources relied on: supplied materials (with type), named reports,
public databases, regulatory filings, linked coverage. Each entry carries a
*location* — document + page / section / heading / anchor — not just a source
name, so a summarizer or human corrector can jump to the exact spot. This is
not academic citation — it is the verifiability chain made visible. A clean,
located sources block raises the perceived authority of the whole piece.

## What to omit

- Adjective stacks with no referent ("innovative", "leading", "industry-first"
  unless sourced).
- Vague metrics ("thousands of customers") with no number.
- Founder vision quotes that state no verifiable fact.
- Filler transitions ("在当今快速变化的时代…").
- Duplicate restatements of the same fact in different wording.

Each of these either gets dropped by the summarizer or, worse, becomes the
seed of a hallucination. Cut them at draft time.
